Redis Performance Improvements in 2024

February 15, 2024

by Antirez

In this article, I'll discuss the major performance improvements we've implemented in Redis over the past year. These optimizations focus on throughput, latency, and memory efficiency.

Memory Optimization

We've completely rewritten the memory allocator to reduce fragmentation. The new implementation uses a buddy system that significantly improves cache locality and reduces the number of system calls.

Initial benchmarks show a 15-20% improvement in throughput for typical workloads, with even larger gains for write-heavy scenarios.

Concurrency Improvements

The threading model has been redesigned to better utilize modern multi-core processors. We've moved away from a single event loop to a more sophisticated approach that balances workload across cores.

The key insight was that different types of operations benefit from different threading strategies. Read-heavy operations can be parallelized more aggressively, while write operations require careful coordination.

Conclusion

These improvements represent years of careful tuning and benchmarking. We're excited to share them with the community and look forward to your feedback.
